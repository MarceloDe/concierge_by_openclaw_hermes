Diagnosis — Current State

What the multi-model review found in the repo and where the critical gaps are.

3-Model Consensus — Critical Findings
GPT-5.5OpenClaw/Hermes are agent harnesses (chat-channel-first runtimes), not API-first web backends. Exposing them directly to the frontend is the #1 architectural risk — they lack request/response guarantees, error contracts, and auth layers needed for a web API.
ClaudeThe repo is missing async task handling. Long LLM calls under load will hit load-balancer timeouts (30–60s). Without task IDs + background processing, the frontend will get dropped connections in production.
GeminiThere's no RAG pipeline yet. For insurance use cases, the LLM answering from pretraining will hallucinate policy specifics, coverage rules, and claim procedures. A retrieval layer over your curated DB is required even for MVP.

Architecture Laws (all 3 models agree — non-negotiable)
  • 1. FastAPI is the ONLY frontend entrypoint. No frontend ever calls OpenClaw or Hermes directly.
  • 2. LangGraph owns state. All conversation flow, node routing, and memory injection happen in LangGraph nodes — not in OpenClaw's internal planner.
  • 3. OpenClaw/Hermes are workers, not orchestrators. They receive a tightly-scoped task envelope and return a structured result. They cannot initiate, branch, or call the frontend.
  • 4. Self-improving skills are Phase 2. For MVP, OpenClaw skill-building is logged but not auto-applied. Human review required before any skill changes production behavior.
  • 5. All LLM answers must be grounded. The retrieval node runs before the LLM node. No insurance-domain answer can be generated without a retrieval context.

FastAPI Service Layer

P0

Create the web API that the frontend will call. This is the public surface — define it clearly and stabilize it before building anything else.

project/api/main.py — minimal MVP entrypoint
from fastapi import FastAPI, Depends, HTTPException, BackgroundTasks
from fastapi.middleware.cors import CORSMiddleware
from fastapi.responses import StreamingResponse
from pydantic import BaseModel
from typing import Optional
import asyncio, uuid

app = FastAPI(title="Wefella Concierge API", version="0.1.0")

app.add_middleware(CORSMiddleware,
    allow_origins=["https://wefella.vercel.app"],  # production domain
    allow_origins_dev=["http://localhost:3000"],    # dev only
    allow_credentials=True, allow_methods=["*"], allow_headers=["*"])

# --- Request/Response contracts (stabilize these early) ---
class ChatRequest(BaseModel):
    message: str
    session_id: Optional[str] = None   # None → new session
    user_id: str                        # required for HIPAA audit log

class ChatResponse(BaseModel):
    session_id: str
    task_id: str
    status: str                         # "streaming" | "queued" | "done"

class TaskStatus(BaseModel):
    task_id: str
    status: str                         # "running" | "done" | "error"
    result: Optional[str] = None

# --- Endpoints ---
@app.post("/api/chat", response_model=ChatResponse)
async def chat(req: ChatRequest, bg: BackgroundTasks):
    session_id = req.session_id or str(uuid.uuid4())
    task_id = str(uuid.uuid4())
    bg.add_task(run_graph_task, task_id, session_id, req.message, req.user_id)
    return ChatResponse(session_id=session_id, task_id=task_id, status="queued")

@app.get("/api/chat/stream/{task_id}")
async def stream(task_id: str):
    return StreamingResponse(token_stream(task_id), media_type="text/event-stream")

@app.get("/api/chat/status/{task_id}", response_model=TaskStatus)
async def task_status(task_id: str):
    # fetch from Redis/in-memory task registry
    ...

@app.get("/api/health")
async def health(): return {"status": "ok", "version": "0.1.0"}
Why this pattern?

Claude Opus 4.8 strongly recommended the POST → task_id → SSE stream pattern rather than a single blocking SSE endpoint. Reason: LLM runs frequently exceed 30–60s, which kills most load balancers. The task_id pattern lets the frontend:

  • • Reconnect to the stream if the connection drops without losing the run
  • • Poll status as a fallback if SSE fails
  • • Display "thinking…" UI state while the background task runs

LangGraph Orchestrator — Node Design

P0

Define the explicit state machine. Every conversation flows through named nodes. No "magic" routing — all decisions are logged and auditable.

project/graph/concierge_graph.py
from langgraph.graph import StateGraph, END
from langgraph.checkpoint.sqlite import SqliteSaver
from typing import TypedDict, Annotated, List
import operator

# --- State schema ---
class ConciergeState(TypedDict):
    messages: Annotated[List[dict], operator.add]  # conversation history
    session_id: str
    user_id: str
    intent: str                # classified intent from intake node
    retrieved_context: str     # RAG chunks injected before LLM call
    citations: List[str]       # source refs for grounding
    worker_task: dict          # optional task envelope for OpenClaw
    worker_result: dict        # optional result from OpenClaw
    final_response: str

# --- Nodes (each is a pure async function) ---
async def intake_guard(state: ConciergeState) -> ConciergeState:
    """PHI scrub + topic boundary check. Rejects out-of-scope requests."""
    ...

async def classify_intent(state: ConciergeState) -> ConciergeState:
    """Classify: plan_inquiry | claim_status | procedure_prep | 
       prescription | escalate | off_topic"""
    ...

async def retrieve_context(state: ConciergeState) -> ConciergeState:
    """RAG over insurance DB. REQUIRED before any LLM generation."""
    ...

async def generate_response(state: ConciergeState) -> ConciergeState:
    """LLM synthesis — ONLY from retrieved_context. Never from pretraining alone."""
    ...

async def worker_delegate(state: ConciergeState) -> ConciergeState:
    """Optional: delegate to OpenClaw/Hermes worker via adapter.
    Only triggers if intent requires tool execution."""
    ...

async def format_and_cite(state: ConciergeState) -> ConciergeState:
    """Attach citations, format for frontend, write audit log entry."""
    ...

# --- Routing ---
def route_after_classify(state: ConciergeState) -> str:
    if state["intent"] == "off_topic": return "intake_guard"    # redirect
    if state["intent"] == "escalate":  return END               # human handoff
    return "retrieve_context"

def route_after_retrieve(state: ConciergeState) -> str:
    if state.get("worker_task"): return "worker_delegate"
    return "generate_response"

# --- Build graph ---
def build_graph(checkpointer):
    g = StateGraph(ConciergeState)
    g.add_node("intake_guard", intake_guard)
    g.add_node("classify_intent", classify_intent)
    g.add_node("retrieve_context", retrieve_context)
    g.add_node("generate_response", generate_response)
    g.add_node("worker_delegate", worker_delegate)
    g.add_node("format_and_cite", format_and_cite)

    g.set_entry_point("intake_guard")
    g.add_edge("intake_guard", "classify_intent")
    g.add_conditional_edges("classify_intent", route_after_classify)
    g.add_conditional_edges("retrieve_context", route_after_retrieve)
    g.add_edge("generate_response", "format_and_cite")
    g.add_edge("worker_delegate", "generate_response")
    g.add_edge("format_and_cite", END)

    return g.compile(checkpointer=checkpointer)

This is the intelligence layer. The curated insurance use cases, claims, and solutions database is what makes Wefella authoritative. The LLM must only synthesize from retrieved evidence — never from pretraining alone for domain-specific facts.

⚠️ Insurance RAG — Critical Rules (GPT + Claude + Gemini all agree)
  • • Every claim-specific answer must include a citation to source document. If the retriever returns nothing, the LLM must say "I don't have verified information on that" — not hallucinate.
  • • Use MMR retrieval (max marginal relevance), not cosine similarity alone — insurance answers often require multiple non-redundant chunks.
  • • Insurance docs have tables and formatted clauses. Use chunk_overlap=150+ to avoid splitting mid-clause.
  • • Add a confidence threshold: if top retrieved doc score < 0.7, trigger escalation rather than a low-confidence answer.
  • • For MVP: Chroma (local). For production: pgvector (Postgres) with your existing infra.

Persistence — Sessions, Checkpoints, Audit Log

P0

Persistence is the difference between a chatbot and a concierge. Sessions must survive restarts. Audit logs are required for HIPAA compliance and insurance regulatory review (NAIC AI governance).

project/persistence/store.py
from langgraph.checkpoint.sqlite import SqliteSaver
# Production: use langgraph.checkpoint.postgres import PostgresSaver
import sqlite3, json, datetime

# --- LangGraph session checkpointing ---
def get_checkpointer(db_path: str = "./data/sessions.db"):
    """Returns a checkpointer. Swap for PostgresSaver in production."""
    conn = sqlite3.connect(db_path, check_same_thread=False)
    return SqliteSaver(conn)

# --- Audit log (HIPAA + NAIC compliance) ---
class AuditLogger:
    """Write-once audit trail. Every LLM call logged with: 
       user_id, session_id, intent, retrieved_sources, 
       llm_response_hash, timestamp, worker_used."""

    def __init__(self, db_path: str = "./data/audit.db"):
        self.conn = sqlite3.connect(db_path, check_same_thread=False)
        self._init_schema()

    def _init_schema(self):
        self.conn.execute("""
            CREATE TABLE IF NOT EXISTS audit_log (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                timestamp TEXT NOT NULL,
                user_id TEXT NOT NULL,
                session_id TEXT NOT NULL,
                intent TEXT,
                message_hash TEXT,       -- SHA-256 of user message (NOT raw PHI)
                retrieved_sources TEXT,  -- JSON array of citation refs
                response_hash TEXT,      -- SHA-256 of LLM response
                worker_used TEXT,        -- "none" | "openclaw" | "hermes"
                confidence_score REAL,
                escalated INTEGER DEFAULT 0
            )
        """)

    def log(self, **kwargs):
        kwargs["timestamp"] = datetime.datetime.utcnow().isoformat()
        cols = ", ".join(kwargs.keys())
        vals = tuple(
            json.dumps(v) if isinstance(v, list) else v 
            for v in kwargs.values()
        )
        self.conn.execute(f"INSERT INTO audit_log ({cols}) VALUES ({','.join('?'*len(vals))})", vals)
        self.conn.commit()

Worker Adapter — OpenClaw / Hermes Interface

P1

The adapter pattern lets you start with a MockWorker (returns deterministic test responses), then swap in OpenClaw or Hermes behind a feature flag — without changing the LangGraph graph nodes.

project/workers/adapter.py
from abc import ABC, abstractmethod
from typing import Optional
import os

# --- Abstract contract ---
class WorkerAdapter(ABC):
    @abstractmethod
    async def execute(self, task: dict) -> dict:
        """
        task envelope:
          {
            "task_id": str,
            "intent": str,           # e.g. "claim_status_lookup"
            "payload": dict,         # intent-specific data
            "context": str,          # RAG context already retrieved
            "session_id": str,
          }
        returns:
          {
            "success": bool,
            "result": str,           # worker's answer
            "skill_used": str,       # which OpenClaw skill was invoked
            "confidence": float,
            "error": Optional[str]
          }
        """
        ...

# --- Phase 1 MVP: Mock worker (always available, deterministic for testing) ---
class MockWorker(WorkerAdapter):
    async def execute(self, task: dict) -> dict:
        return {"success": True, "result": f"[MOCK] Processed: {task['intent']}",
                "skill_used": "mock", "confidence": 1.0, "error": None}

# --- Phase 2: OpenClaw worker ---
class OpenClawWorker(WorkerAdapter):
    def __init__(self, openclaw_url: str, api_key: str):
        self.url = openclaw_url
        self.api_key = api_key

    async def execute(self, task: dict) -> dict:
        # POST task to OpenClaw's MCP/channel endpoint
        # OpenClaw returns structured result
        ...

# --- Factory: controlled by environment variable / feature flag ---
def get_worker() -> WorkerAdapter:
    worker_type = os.getenv("WORKER_BACKEND", "mock")  # "mock" | "openclaw" | "hermes"
    if worker_type == "openclaw":
        return OpenClawWorker(os.getenv("OPENCLAW_URL"), os.getenv("OPENCLAW_KEY"))
    return MockWorker()

Streaming + Frontend API Contract

P0
project/api/streaming.py — SSE token stream generator
import asyncio, json
from typing import AsyncGenerator

# In-memory task registry (swap for Redis in production)
task_registry = {}

async def run_graph_task(task_id: str, session_id: str, message: str, user_id: str):
    """Background task: runs LangGraph, writes tokens to registry."""
    task_registry[task_id] = {"status": "running", "tokens": [], "done": False}
    try:
        graph = build_graph(get_checkpointer())
        config = {"configurable": {"thread_id": session_id}}

        async for event in graph.astream_events(
            {"messages": [{"role": "user", "content": message}],
             "session_id": session_id, "user_id": user_id},
            config=config, version="v2"
        ):
            if event["event"] == "on_chat_model_stream":
                token = event["data"]["chunk"].content
                task_registry[task_id]["tokens"].append(token)

        task_registry[task_id]["status"] = "done"
        task_registry[task_id]["done"] = True
    except Exception as e:
        task_registry[task_id]["status"] = "error"
        task_registry[task_id]["error"] = str(e)

async def token_stream(task_id: str) -> AsyncGenerator[str, None]:
    """SSE generator. Frontend connects here to read tokens."""
    cursor = 0
    while True:
        reg = task_registry.get(task_id)
        if not reg:
            yield f"data: {json.dumps({'error': 'task_not_found'})}

"
            return

        tokens = reg["tokens"][cursor:]
        for token in tokens:
            yield f"data: {json.dumps({'token': token})}

"
        cursor += len(tokens)

        if reg["done"]:
            yield f"data: {json.dumps({'done': True, 'status': reg['status']})}

"
            return

        await asyncio.sleep(0.05)  # 50ms poll interval

Security, HIPAA & Compliance Guardrails

P0
HIPAA Requirements
  • • BAA required with OpenAI/Anthropic API (API tier, not consumer products)
  • • Never store raw PHI in conversation logs — hash user messages in audit log
  • • Encrypt data at rest (SQLite/Postgres with encryption)
  • • Role-based access: user vs admin endpoints
  • • Session tokens expire (<24h for healthcare data)
FastAPI Security Checklist
  • • JWT auth on all non-health endpoints (OAuth2 + JWT)
  • • Rate limiting: slowapi or Redis-backed (max 20 req/min/user)
  • • CORS: production domain only in allow_origins
  • • Input validation: Pydantic models on all request bodies
  • • HTTPS-only in production (terminate at load balancer)

from fastapi import Security, HTTPException, status
from fastapi.security import OAuth2PasswordBearer
from slowapi import Limiter
from slowapi.util import get_remote_address
import jwt, os

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")
limiter = Limiter(key_func=get_remote_address)

async def get_current_user(token: str = Security(oauth2_scheme)):
    try:
        payload = jwt.decode(token, os.getenv("JWT_SECRET"), algorithms=["HS256"])
        return payload
    except jwt.ExpiredSignatureError:
        raise HTTPException(status_code=401, detail="Token expired")
    except jwt.InvalidTokenError:
        raise HTTPException(status_code=401, detail="Invalid token")

You are implementing the backend for Wefella, a guided AI concierge for healthcare 
plan selection, plan use, prescriptions, and procedure preparation. The backend

3. OpenClaw/Hermes are workers: they receive a typed task envelope, return a structured result.

   Never answer from LLM pretraining alone for domain-specific facts.
5. Write a write-once audit log for every LLM call: timestamp, user_id, intent,
